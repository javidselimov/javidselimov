## 🚀 Featured Project — reactivite

<a href="https://reactivite.vercel.app">
  <img src="https://media.daily.dev/image/upload/s--49WXl71n--/f_auto/v1781888254/posts/BY5JamQ5z?_a=BAMAMiWQ0" alt="reactivite — build React apps lightning fast" width="100%"/>
</a>

**[reactivite](https://reactivite.vercel.app)** — scaffold production-ready React apps in **one command**.

```bash
npm create reactivite@latest my-app
# ✓ React + Vite + Tailwind in ./my-app
```

Every React project starts the same way: spin up Vite, wire TypeScript, add ESLint + Prettier, configure Tailwind, pick a folder structure… an hour of boilerplate before line one of real code. **reactivite collapses that into a single command** — zero config, sane production-ready defaults.

**What you get out of the box:**
- ⚡ **Vite** — instant dev server + HMR
- 🔷 **TypeScript** wired in from the start
- 🎨 **Tailwind CSS** + optional **shadcn/ui**
- 🧹 **ESLint + Prettier** pre-configured
- 📁 Sane folder structure following industry patterns
- 📦 Optimized, deployment-ready production build

It's a small CLI toolkit — same "one command, production defaults" philosophy across different starting points:

| Command | Scaffolds |
|---|---|
| `npm create reactivite@latest` | React + Vite + Tailwind (+ optional shadcn/ui, TS) |
| `npm create dolcevite@latest` | Batteries-included React: Zustand + TanStack Query + Ant Design |
| `npm create @ubuligan/mcp-server@latest` | MCP server with auth, rate-limiting, logging, deploy config |
| `npx @ubuligan/create-app-skill` | A Claude Code skill that scaffolds Next.js / React apps |

🌐 **[Site & docs](https://reactivite.vercel.app)** · 📦 **[Packages](https://reactivite.vercel.app/packages)** · 🧰 **[CLI reference](https://reactivite.vercel.app/cli)** · 🐙 **[GitHub](https://github.com/javidselimov)**
